## 🎯 조직별 분석 요구사항 정의

### 1. **병원 전체 분석**
```
📊 병원장용 대시보드
├── 전체 협업 만족도 지수
├── 부문별 성과 비교
├── 주요 이슈 TOP 5
├── 연도별 트렌드 분석
└── 개선 우선순위 권고사항
```

### 2. **부문별 분석** 
```
📋 부문장용 리포트 (간호부문, 진료부문, 관리부문, 기획부문)
├── 부문 내 부서별 성과 비교
├── 타 부문과의 협업 평가
├── 부문 특화 이슈 분석
├── 우수 사례 및 개선 필요 영역
└── 부문별 액션 플랜 제안
```

### 3. **부서별 분석**
```
📝 부서장용 상세 리포트
├── 부서 협업 종합 점수
├── 타 부서로부터 받은 평가
├── 타 부서에게 한 평가  
├── 부서 내 Unit별 성과
├── 구체적 피드백 내용
└── 맞춤형 개선 방안
```

### 4. **Unit별 분석**
```
🔍 Unit Manager용 실행 리포트
├── Unit 협업 성과 지표
├── 개별 피드백 상세 내용
├── 감정 분석 및 키워드 분석
├── 품질 점수 및 재검토 사항
└── 즉시 실행 가능한 개선 사항
```

## 🎛️ 권한별 접근 시스템 설계

### 🏥 **병원장 (Level 1)**
- **접근 범위**: 전체 병원
- **제공 정보**: 
  - 병원 전체 협업 지수
  - 부문별 성과 순위
  - 주요 이슈 대시보드
  - 연도별 비교 분석

### 🏢 **부문장 (Level 2)**
- **접근 범위**: 해당 부문 + 타 부문과의 협업 데이터
- **제공 정보**:
  - 부문 내 세부 성과 분석
  - 타 부문과의 협업 현황
  - 부문별 특화 개선 방안

### 👥 **부서장 (Level 3)**  
- **접근 범위**: 해당 부서 + 관련 협업 부서
- **제공 정보**:
  - 부서 상세 성과 분석
  - 받은 평가 vs 한 평가 비교
  - Unit별 세부 분석

### ⚙️ **Unit Manager (Level 4)**
- **접근 범위**: 해당 Unit만
- **제공 정보**:
  - Unit 개별 피드백 전체
  - 실행 가능한 개선 사항
  - 품질 검증 결과

### 🖥️ **웹 대시보드 구성**

#### **공통 요소**
- 로그인 시스템 (권한별 접근 제어)
- 실시간 데이터 업데이트
- 인터랙티브 차트 및 필터링
- 드릴다운 기능 (병원 → 부문 → 부서 → Unit)

#### **메인 대시보드 레이아웃**
```
┌─────────────────────────────────────────┐
│ 🏥 서울아산병원 협업 분석 대시보드        │
├─────────────────────────────────────────┤
│ 📊 KPI 카드 (만족도, 문제률, 개선률)     │
├─────────────────────────────────────────┤

├─────────────────────────────────────────┤

└─────────────────────────────────────────┘
```

### 📄 **PDF 리포트 템플릿**

#### **병원장용 Executive Summary (4페이지)**
1. 페이지 1: 전체 성과 요약 및 핵심 지표
2. 페이지 2: 부문별 성과 비교 및 순위
3. 페이지 3: 주요 이슈 및 트렌드 분석
4. 페이지 4: 전략적 개선 권고사항

#### **부문장용 Departmental Report (6페이지)**
1. 페이지 1: 부문 성과 요약
2. 페이지 2: 부문 내 부서별 상세 분석
3. 페이지 3: 타 부문과의 협업 현황
4. 페이지 4: 부문 특화 이슈 분석
5. 페이지 5: 우수 사례 및 베스트 프랙티스
6. 페이지 6: 액션 플랜 및 개선 로드맵

#### **부서장/Unit Manager용 Operational Report (8페이지)**
1. 페이지 1: 부서/Unit 성과 요약
2. 페이지 2: 받은 평가 상세 분석
3. 페이지 3: 한 평가 상세 분석  
4. 페이지 4: 감정 분석 및 키워드 분석
5. 페이지 5: 품질 검증 결과
6. 페이지 6: 개별 피드백 전문
7. 페이지 7: 즉시 실행 가능한 개선 사항
8. 페이지 8: 모니터링 지표 및 후속 조치

## 🔧 기술 구현 방안

### **대시보드 기술 스택**
- **Frontend**: Streamlit 또는 Dash (Python 기반)
- **Backend**: FastAPI + SQLite/PostgreSQL
- **Visualization**: Plotly, Seaborn, Matplotlib
- **Authentication**: 역할 기반 접근 제어 (RBAC)

### **PDF 생성 기술**
- **Library**: ReportLab + Matplotlib
- **Template**: HTML → PDF 변환 (WeasyPrint)
- **Automation**: 정기 리포트 자동 생성 및 이메일 발송

### **데이터 파이프라인**
```
원본 Excel → 전처리 → AI 분석 → 조직별 집계 → 대시보드 DB → 실시간 시각화
                ↓
         PDF 리포트 생성 → 권한별 배포
```

## 📅 구현 단계별 계획

### **Phase 1: 기본 분석 시스템 (2주)**
- 조직별 데이터 집계 로직 구현
- 기본 통계 및 지표 계산
- 간단한 대시보드 프로토타입

- 감정 분석 기반 조직별 인사이트

### **Phase 3: 대시보드 및 리포트 (3주)**
- 권한별 대시보드 구현
- PDF 리포트 자동 생성
- 사용자 인터페이스 최적화

### **Phase 4: 배포 및 운영 (1주)**
- 시스템 배포 및 사용자 교육
- 모니터링 및 피드백 수집
- 지속적 개선 방안 수립
